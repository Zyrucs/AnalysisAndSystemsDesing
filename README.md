### Folder Description

- **diagrams/**
  Contains the system representation diagrams such as the system structure diagram and the operational flowchart.

- **charts/**
  Includes charts generated from the student survey used for primary data collection.

- **data/**
  Raw data collected from the survey conducted with university students.

- **report/**
  Final PDF document containing the full systems analysis.

---

# Data Collection Methods

The analysis incorporates **primary data collection** through the following methods:

- **Student survey** to understand digital access conditions and device availability.
- **System observation** to identify operational processes and interactions.
- **Process documentation** to map how resources and technical support has to be managed.

The collected data was analyzed and visualized using charts to identify patterns related to digital accessibility among students.

---

# System Analysis Components

The analysis includes the following elements:

- **System boundary definition**
- **Identification of system inputs and outputs**
- **Element relationship mapping**
- **Operational process flow**
- **Survey data analysis**
- **System performance evaluation**

Diagrams were created to represent:

- The structure and components of the system
- The operational flow of resource allocation
- Relationships between stakeholders and system elements

---

# Key Findings

The analysis highlights several important aspects of the Digital Inclusion Bridge System:

- Device availability and internet access remain critical factors for students'learning.
- Technical support plays an essential role in maintaining system usability.
- Proper resource management and monitoring can significantly improve digital inclusion outcomes.
